# ADR-001: Wrap the Community VPC Module

## Status

Accepted

## Context

We need a reusable VPC module that provides public and private subnets across multiple availability zones with optional NAT Gateway support. The `terraform-aws-modules/vpc/aws` community module is mature, well-tested, and widely adopted.

## Decision

Wrap the community VPC module rather than implementing VPC resources directly. This gives us:
- A stable, battle-tested foundation maintained by the community
- An opinionated interface that exposes only the variables our consumers need
- The ability to add organization-specific defaults (tags, naming, etc.) in one place
- Easy upgrades when the upstream module releases new features

## Consequences

- Consumers get a simpler, more focused interface
- Upgrades to the upstream module are isolated to this wrapper
- We inherit the community module's behavior and any breaking changes on upgrade
